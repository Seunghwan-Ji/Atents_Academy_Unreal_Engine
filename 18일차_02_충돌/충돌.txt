1. BP_Sword -> 컴포넌트 추가 -> 검색 키워드: col -> Capsule Collision 선택.
- 이름 예시: Capsule
2. Capsule -> 디테일 -> 소켓 -> 부모 소켓 -> center.
- 캡슐 영역이 center 소켓에 위치함.
3. 셰이프 -> 캡슐 절반 높이: (예: 50)
- 충돌 영역이 타이트하면 공격 애니메이션마다 충돌 운이 변동하는 경우가 많기 때문에 유저 편의성을 위해 충돌 영역을 여유롭게 하는편.
4. 콘텐츠 드로어 -> 추가 -> 블루프린트 클래스 -> 액터.
- 이름 예시: BP_BoxCollision
5. BP_BoxCollision -> 컴포넌트 추가 -> 검색 키워드: col -> Box Collision 선택.
- 이름 예시: Box
6. Box 컴포넌트를 루트 컴포넌트로 지정.
7. Box -> 디테일 -> 트랜스폼 -> 스케일 조정 -> 예: 2, 2, 3
8. 렌더링 -> 게임에서 숨김 체크 해제.
- 표시 옵션은 뷰포트에서 박스 영역을 표시할지 여부이고, 게임에서 숨김 옵션은 플레이시 박스 영역을 표시할지 여부임.
10. 디테일 -> 콜리전 프리셋 (프리셋: 미리 설정하다) -> Custom, 오브젝트 타입 -> WorldStatic.
- 콜리전 반응 옵션:
- 무시: 충돌을 무시함, 충돌처리 우선순위1
	- 오버랩(겹침): 충돌했을 때 이벤트 호출 가능, 우선순위2
	- 블록(막음): 충돌 시 막힘, 이벤트 호출 가능, 우선순위3
	- 예를들어 두 충돌체가 충돌했을 때 한 쪽은 오버랩이고 한 쪽은 블록이면 우선순위가 더 높은 오버랩 판정이 발생함.
- 오브젝트 타입 옵션:
	- 오브젝트 타입을 설정하면 다른 충돌체가 자신과 충돌했을 때 어떤 오브젝트 타입과 충돌되었는지 판단하는데 참조됨.
- 콜리전 활성화됨 옵션:
	- No Collision: 이 충돌체는 충돌 검사 안함.
	- Query Only: 영역에 대해서만 충돌 검사.
	- Physics Only: 물리에 대해서만 충돌 검사.
	- Collision Enabled: 영역과 물리 모두 충돌 검사.
11. 컴포넌트 추가 -> 검색 키워드: col -> Sphere Collision 선택.
- 이름 예시: Sphere
12. Sphere -> 디테일 -> 콜리전 프리셋 -> OverlapAll, 게임에서 숨김 체크 해제.
13. 블루프린트 뷰포트에서 Sphere 를 박스옆에 배치.
14. Box -> 디테일 -> 이벤트 -> 컴포넌트 오버랩 시작 시 선택.
15. 이벤트 그래프에서 수업 시간에 작성한 로직 확인.
