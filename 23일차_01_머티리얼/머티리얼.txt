# 머티리얼
1. 콘텐츠 드로어 -> Meshes -> Characters -> Mannequin -> Materials -> M_Body 더블 클릭 -> 머티리얼 그래프.
- M_Body: 머티리얼 타입.
- BodyColor 노드: M_Body 머티리얼 에셋의 배경색을 설정함.
- Mask 노드: Mannequin -> Textures -> T_Body_Mask 텍스쳐 에셋을 사용하는 노드.
	- 뷰포트 하단 세 번째 평면 도형 버튼을 클릭하면 이 텍스쳐 모양대로 보임.
	- R 출력핀: 이 텍스쳐에서 빨간색 영역들만 따로 뽑아서 처리.
	- G 출력핀: 이 텍스쳐에서 초록색 영역들만 따로 뽑아서 처리.
	- B 출력핀: 이 텍스쳐에서 파란색 영역들만 따로 뽑아서 처리.
	- 마스킹 테이프를 사용하는 듯한 원리.
- NormalMap 노드: Mannequin -> Textures -> T_Body_Normal 에셋을 사용하는 노드.
	- 표면에 부분적으로 굴곡(요철 정보)을 설정함. (뷰포트에서 울퉁불퉁한 부분이 이 설정 때문임)
	- 굴곡 설정은 메시에서 하는게 적합해 보이지만 비용이 비쌈.
	- 굴곡처럼 보이게하는 착시 효과이며, 가까이서 보면 표면이 매끈해보임.
- Emissive 노드: 머티리얼 표면에 빛이 나는 효과를 줄 수 있음. (플래쉬 효과)
	- Emissive 노드를 선택하고 디테일에서 RGB 값을 변경하면서 체감해 볼 수 있음.
- 이 M_Body 가 메시에 자동으로 적용됨.

# 머티리얼 인스턴스
1. 콘텐츠 드로어 -> Meshes -> Characters -> Mannequin -> Materials -> MI_Logo 더블 클릭.
- 기본적으로 머티리얼 인스턴스는 머티리얼의 설정을 따름. 그러므로 MI_Logo 는 M_Body 의 설정을 따를 수 있음.
	- MI_Logo -> 디테일 -> 파라미터 그룹을 보면 M_Body -> 파라미터의 설정 값들이 들어있음.
2. M_Body -> 머티리얼 그래프에서 PlasticOverride 노드 살펴보기.
- MI_Logo 의 설정 값이 정해지는 노드.
- 디폴트 값이 체크되어 있으면 MI_Logo 는 True 핀으로 입력되는 MI_Logo 전용 설정 값이 적용되고, 체크 해제되어 있으면 False 핀으로 입력되는 M_Body 의 설정 값을 따르게됨.
	- 이 논리 값 체크 여부가 M_Body -> 파라미터 탭 -> PlasticOverride 항목임.
- PlasticOverride 체크 여부는 MI_Logo -> 디테일 -> 파라미터 그룹 -> Global Static Switch 에서도 설정할 수 있음.
	- PlasticOverride 항목 오른쪽은 이 오버라이드를 반영할 지 여부임. 체크하면 오버라이드를 반영함.
- Global Texture Parameter Values 항목에서 MI_Logo 전용 굴곡 텍스쳐를 지정할 수 있음. (기존 굴곡 텍스쳐는 M_Body 전용)
- 이처럼 머티리얼 인스턴스는, 베이스가 되는 머티리얼이 있고 베이스 머티리얼에서 전달하는 파라미터중 필요한 데이터만 변경하고 싶은 경우 사용함.
- MI_Logo 뷰포트에서 투명한 부분이 있는데, 이는 M_Body -> 머티리얼 그래프 -> PlasticOverride 노드에서 True 입력핀으로 들어오는 함수 내부의 Texture Sample 노드 -> 로고 원 바깥 부분 A 값(불투명도)이 0으로 설정되어 있기 때문임.
- 몽타주 애니메이션이 시퀀스 애니메이션에 의존하듯 머티리얼 인스턴스도 머티리얼에 의존함.
	- MI_Logo -> 디테일 -> 일반 -> 부모 항목을 보면 M_Body 인 것을 알 수 있음.
	- 머티리얼이 없으면 머티리얼 인스턴스를 사용할 수 없음.
