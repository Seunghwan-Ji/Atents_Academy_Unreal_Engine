1. 언리얼 엔진의 모든 에셋은 기본적으로 확장자가 .uasset 임.
2. 어떤 프로젝트의 에셋을 다른 프로젝트로 옮길때는 콘텐츠 드로어에서 에셋이 포함된 폴더(예: 콘텐츠)를 우클릭 -> 이주 -> 이동할 경로 선택(예: 다른 프로젝트의 콘텐츠 폴더).
3. 블루프린트의 기본적인 클래스 계층 구조: 콘텐츠 드로어에서 우클릭 -> 블루프린트 클래스를 눌러 구조를 읽어보기.
- 기본적으로 오브젝트 클래스에서 파생된 클래스들임.
4. 캐릭터 클래스
- 콘텐츠 드로어 -> 추가 -> 블루프린트 -> 캐릭터 -> 이름 예시: BP_Character
- 캡슐 컴포넌트 - 기본적으로 캐릭터의 영역을 표시하며, 그 영역이 충돌 영역이기도 함.
	- 디테일 -> 셰이프 -> 캡슐 절반 높이 -> 91.5 (메시 에셋의 높이와 맞추는 작업)
	- 화살표 컴포넌트: 캐릭터가 바라보는 방향을 가짐.
	- 메시 컴포넌트: 캐릭터의 외형 에셋 및 트랜스폼 등을 설정할 수 있음. 외형은 메시 형태에 머터리얼을 입힘.
		- 디테일 -> 메시 -> 스켈레탈 메시 에셋 -> SKM_Mannequin
		- 디테일 -> 트랜스폼 -> Z축 위치 -> 91.5 (캡슐 위치와 맞추는 작업)
		- 디테일 -> 트랜스폼 -> Z축 회전 -> -90 (바라보는 방향과 일치시키는 작업)
- 캐릭터 무브먼트: 캐릭터의 물리작용에 관여하는 기능.
- 캡슐 컴포넌트는 트랜스폼과 관련있기 때문에 씬 컴포넌트에 속하고, 캐릭터 무브먼트는 캐릭터의 기능과 관련있기 때문에 액터 컴포넌트에 속함.
- 또한 캡슐 컴포넌트는 루트 컴포넌트임. 루트 컴포넌트는 씬 컴포넌트를 가진 컴포넌트중 최상위 컴포넌트를 말함.
- 기본적으로 씬 컴포넌트를 가진 부모 컴포넌트의 트랜스폼 값이 변경되면 씬 컴포넌트 기능을 가진 자식 컴포넌트들도 따라서 바뀜.
- 루트 컴포넌트는 디테일 -> 트랜스폼에서 회전값 및 위치값을 설정할 수 없음.
- 레벨에 배치하고 레벨 상의 뷰포트 -> 디테일에서만 조정할 수 있음.
- 에를들어 객체가 매번 같은 위치에 스폰되는 것을 문제로 삼음.
5. 블루프린트 클래스 내 이벤트 그래프 탭은 그 클래스에 대해 프로그래밍하는 공간임. 좌측 상단의 컴파일 버튼을 누르면 작성한 로직을 레벨에 반영할 수 있고, 클래스의 객체를 레벨에 배치하고 뷰포트를 실행하면 블루프린트에서 작성한 로직이 실행됨.
6. 이벤트 그래프 탭의 노드에서 화살표를 핀이라고 부르며, 왼쪽 핀은 입력핀, 오른쪽 핀은 출력핀이라고 부름.
- 이름 없는 하얀 핀은 실행핀이라고 부름.
7. 핀은 같은색의 핀끼리 연결해야함.
8. 이벤트 그래프 탭의 내 블루프린트 탭에서 변수 노드 등을 정의할 수 있음.
9. 만든 캐릭터 클래스는 콘텐츠 드로어에서 클래스를 드래그해 레벨에 배치하여 객체를 생성할 수 있음.
